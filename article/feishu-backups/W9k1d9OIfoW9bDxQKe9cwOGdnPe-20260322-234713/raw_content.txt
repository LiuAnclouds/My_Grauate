2. 阅读主线
本文最核心的判断可以压缩为一句话：

在当前主流 graph benchmark 上，classic GNN 长期显得不如 GT，并不一定是因为 message passing 范式天然更弱，也可能是因为它们长期没有以一个完整、现代、可公平比较的 block 形式参与实验；当这些组件被系统补齐后，classic GNN 可以成为强 baseline，甚至在多个数据集上达到或超过当时的 SOTA GT，同时保持更好的训练效率。
阅读这篇论文时，最值得抓住的是三条主线：
• 问题主线：社区为什么会默认 classic GNN 弱于 GT，这个默认认识是否建立在对称比较上。
• 方法主线：GNN+ 究竟补了哪些部件，使 classic GNN 从“传播层”升级为“完整 block”。
• 证据主线：主文结果、消融、敏感性分析和附录补充，是否真的足以支撑“classic GNN 可以成为强 baseline”这一结论。

三、Question&Study
1. 比较缺口
graph learning 社区里有一个比较稳定的默认判断：在 graph-level task 上，Graph Transformer 往往比 classic GNN 更强，原因通常包括全局注意力更容易处理长程依赖、message passing 更容易受 over-smoothing 和 over-squashing 影响、以及 GT 往往在分子图和长程依赖 benchmark 上表现更好。
这个判断并不是空穴来风，但它有两个比较明显的前提漏洞。
第一个漏洞是，node-level 领域已经出现过一轮比较系统的“重新评估”。相关工作表明，在统一超参数空间和更公平协议下，classic GNN 并不一定输给更新、更复杂的模型。但是这一类结论并没有被系统搬到 graph-level 场景里，尤其没有用一组覆盖较广的数据集来重新检验。
第二个漏洞是，很多比较其实不是“范式公平比较”，而是“配置不对称比较”。GT 在实验里通常已经拥有更完整的 block 设计、更充分的调参空间和更成熟的训练 recipe，而 classic GNN 往往还停留在相对朴素的传播层形态。这样得到的结果，混合了 范式差异 和 工程配置差异，两者并没有被明确区分。
2. 核心主张
把本文的问题压缩之后，可以写成：

在 graph-level 相关 benchmark 上，如果 classic GNN 也被构造成完整、现代、可公平比较的 deep block，它们是否仍然只是弱 baseline，还是其实可以成为强 baseline，甚至达到或超过当时的 SOTA GT？
作者给出的回答不是再发明一种全新的图网络，而是提出 GNN+。这套方法保留了 classic message passing 的主体，但把一个现代深层 block 中常见且关键的部件系统接入：边特征、位置编码、归一化、残差、dropout 和 FFN。换句话说，作者的主张是：过去很多对 classic GNN 的评判，实际上是在拿“完整 GT block”和“残缺 GNN layer”作比较。
从贡献性质看，本文的创新更适合分成三层：

因此，本文最准确的阅读方式不是“这又是一个新的 graph model”，而是“这是一篇在 graph benchmark 上重新定义强 baseline 的论文”。
3. 证据边界
本文能支持的结论，其实边界很明确。它能证明的是：在作者覆盖的 benchmark 范围内，在给定的超参数空间和训练协议下，增强后的 classic GNN 确实可以表现得非常强，甚至在多个数据集上进入第一梯队。
但它没有证明以下几件事：
classic GNN 在所有未来任务上都会优于 GT；
GNN+ 为什么有效已经被理论上解释清楚；
六个模块存在一个对所有数据集都统一最优的固定组合；
当前 benchmark 上的结果可以无条件外推到真实复杂应用。
还有一个需要单独记住的边界：这篇论文标题写的是 graph-level tasks，但主文和附录中的实验集合并不全是纯 graph-level 任务，包含若干 inductive node classification benchmark。因此，更精确的表述应该是：本文在“以图架构为核心的一组主流 benchmark”上重新评估了 classic GNN，而 graph-level 任务是其主要叙事重心。

这三个模块共同回答的不是“模型够不够强”，而是“模型能不能稳当地训深”。BN 被放在传播输出之后，用来控制层间分布漂移；Residual 保留一条干净的跨层路径，用来维持梯度和旧信息流；Dropout 则用于缓解图表示学习中的共适应和过拟合。
这组模块最关键的理解是：它们大部分都不是为了“增加表达上限”，而是为了避免 classic GNN 以一种不稳定、难调、难加深的形态参与比较。也就是说，后面如果实验显示这些组件很重要，结论不应被解读成“这些技巧特别神奇”，而应解读成：一个强 baseline 本来就不该缺这些基础设施。
表征加工：FFN
论文中的 FFN 写成：
 
这里的作用和 Transformer block 很像：message passing 负责节点之间的信息交换，但交换之后还需要一次逐节点、逐位置的非线性特征加工。也就是说，GNN+ 中的每一层不再只是“邻域传播层”，而是“传播 + 稳定化 + 特征加工”的完整层。
这一点对 GIN 尤其重要，因为 GIN 本来就强调“聚合后做强非线性判断”；如果缺少 FFN，这条设计路线的表达力实际上是被削弱的。

• 覆盖面：14 个数据集横跨三类 benchmark，而不是只挑容易赢的场景。
• 对称性：classic GNN 和 GT 被放进相近的搜索空间里比较。
• 完整性：不仅报准确率，也报均值方差和训练速度。
如果把这一节压成一句话，它真正要解决的是：后面所有“GNN+ 很强”的结果，到底是模型本身强，还是比较方式终于像样了。

除了主文消融，附录还有四组补充结果值得保留：
• 深度敏感性与 dropout 敏感性（Figure 3 / Figure 4）：Figure 3 显示 GNN+ 在 PATTERN、CLUSTER、PascalVOC-SP 上可以在较宽层数范围内保持稳定表现，但去掉 residual 后超过 8 层会明显退化；Figure 4 则显示最佳 dropout 区间集中在 0.1 ~ 0.2。
• 固定配置结果（Appendix Table 14）：GNN+(fixed) 在 9 个数据集上的结果与 best tuning 非常接近，说明 GNN+ 的有效性并不完全依赖极细致的 dataset-specific tuning。
• 额外 baseline：快线性图分类模型（Appendix Table 15 / Table 16）：GCN+ 在大规模 OGB 分子数据集上的性能明显优于 LDP、HRN，同时保持较合理的训练成本。
• 额外 baseline：缓解 over-smoothing / over-squashing 的专门模型（Appendix Table 17）：GCN+ 在同协议下优于 MTGCN 和 CO-GNN，说明系统增强后的 classic GNN 不一定需要额外引入更复杂的新结构才能取得强结果。
最后还要记住附录 B.5 给出的一个解释框架：static attention problem。作者观察到，在某些 GT（文中以 GraphGPS 为例）里，很多 query node 学到的注意力热点高度相似，注意力几乎总是集中在同几个结构上，而不是随 query node 改变。这会导致模型对局部功能子结构不够敏感。相对地，message-passing GNN 的节点级聚合是 query-specific 的，更容易形成多样化的局部结构编码。这个解释仍然是经验性的，不是严格理论结论，但它为“为什么 classic GNN 在 graph benchmark 上仍然能很强”提供了一个有价值的分析视角。
如果把这一整节压成最后一句话，那么更稳的结论是：GNN+ 之所以成立，不是因为某一个模块神奇，而是因为这些模块共同把 classic GNN 补成了一个结构更完整、训练更稳定、表达更充分的 graph block。
五、Conclusion
1. 核心结论
本文最终建立的核心判断是：

在当前主流 graph benchmark 上，经过系统增强的 classic GNN 可以匹配甚至超过当时的 SOTA GT，并且通常具有更好的训练效率。
2. 主要判断
结合主文结果、消融和附录补充，本文真正支持的判断主要有三点：
• classic GNN 在 graph benchmark 上长期存在被低估的情况。这个“被低估”并不是抽象评价，而是有具体实验形态支撑的：以不完整 block 参与比较，会显著拉低 classic GNN 的表现。
• GT 的优势并不是无条件、绝对性的。尤其在 LRGB 和 OGB 上，增强后的 classic GNN 不仅没有天然掉队，反而在多个数据集上进入第一梯队，并且经常表现出更好的效率。
• 在很多图任务中，系统增强比单纯追逐更复杂的新范式更重要。本文最有说服力的地方，不是提出了某个神奇模块，而是说明“完整而现代的 block 设计”本身就是决定 baseline 强弱的核心因素。
3. 可借鉴点
如果从“这篇论文能给后续研究或做题方式带来什么”来总结，我会更看重下面三点：
• 先把 baseline 做完整，再讨论范式优劣。 这篇论文最值得借鉴的不是某个单模块，而是一种比较原则：如果 baseline 还停留在低配层级，那么后续所有“新模型更强”的判断都可能混入配置偏差。
• 把图模型看成完整 block，而不是只看传播核。 传播规则当然重要，但 PE、Edge、Norm、Residual、Dropout、FFN 这些 block 级设计同样会显著影响最终结论。
• 结果分析要同时看性能、稳定性和效率。 本文最有说服力的地方之一，是没有把比较收缩成单一精度数字，而是连同均值方差、训练速度和附录补证一起看。
4. 局限与改进
本文并没有证明：
classic GNN 在所有未来任务上都一定优于 GT；
GNN+ 强的原因已经被理论上完全解释；
六个模块存在普适最优的固定组合；
当前 benchmark 上的经验结论可以无条件外推到真实复杂应用。
如果再往“后续还可以怎么补”这一步走，比较自然的延伸方向有三类：
• 理论解释还不够闭合。 现在主结论更多仍然建立在系统实证而不是严格理论解释上，尤其是对 long-range dependency、static attention problem 和组件协同机制的说明，还偏经验性。
• 任务范围仍然有限。 当前 benchmark 虽然已经很广，但仍然主要集中在标准图学习数据集上，是否能稳定外推到真实工业图、异构图、动态图等场景，还不能直接下结论。
• 组件交互还有进一步细化空间。 论文已经说明不同模块有分工，但“哪类任务最依赖哪组组件、组件之间是否存在稳定组合规律”仍然值得继续拆。
此外，标题中的 graph-level tasks 与实际 benchmark 集合也存在轻微不完全对齐的问题，因为实验中包含了若干 inductive node classification 数据集。因此，更准确的表述应该是：本文在一组以图架构为核心的主流 benchmark 上，重新评估了 classic GNN 的 baseline 地位。
5. 全文概括
这篇论文最有价值的地方，不是提出了一个全新的图神经网络算子，而是系统说明了：在 graph benchmark 上，classic GNN 之所以长期显得不如 GT，很多时候并不是因为它们在范式上天然更弱，而是因为它们长期没有以一个完整、现代、可公平比较的形态参与实验。GNN+ 通过把 edge、normalization、dropout、residual、FFN 和 PE 统一接入 classic GNN，使 classic GNN 在多个 benchmark 上重新表现出很强的竞争力。本文真正改变的，其实是对“强 baseline 应该如何构建”的理解。
如果把全文再压缩成一句更适合以后快速回看的话，可以记成：
GNN+ 的真正贡献不是发明新传播核，而是把 classic GNN 以完整 modern block 的形态重新放回公平比较中，并据此改写了“强 baseline 应该长什么样”的判断。
